Tasks & Projects

These are the main tasks and projects I was involved in during my internship.

PQC Sandbox

I worked on a PQC sandbox for The National PQC Migration Plan on issuing and testing post-quantum digital certificates . The PQC Sandbox is a simulated Certificate Authority (CA) environment designed to test PQC algorithms without affecting production systems. It implements full PKI components including Root CA, Intermediate CA, certificate profiles, and signing workflows.

Link to sandbox: https://digitalid2.msctrustgate.com/sb-pqc/

  • Issued PQC certificates using KAZ-SIGN (Malaysia's PQC Algorithm by UPM) and ML-DSA.
  • Simulated root, intermediate and end-entity certificate chains.
  • Verified signing and verification flows for PQC algorithms.
Java, MVC, Spring Boot, C++ KAZ-SIGN, ML-DSA and SLH-DSA PQC Sandbox

Benchmarking PQC Algorithm

I was responsible for evaluating the performance of the PQC algorithms used inside the sandbox, mainly focusing on key generation, signing, verification, and output sizes.

  • Ran key generation, signing and verification timing tests.
  • Compared key sizes and signature sizes across algorithms.
  • Documented algorithm differences between KAZ-SIGN and ML-DSA.
  • Summarised the results into simple tables and charts for reporting.

Certificates & Profiles

This section involves configuring certificate structures used in the sandbox, following the PQC certificate issuance requirements. It focuses on mapping PQC algorithms into real PKI profiles while maintaining compatibility with RFC 5280.

  • Creating X.509 profiles for Document Signing, TLS, and Client certificates
  • Configured extensions such as EKU and basic constraints.
  • Helped align PQC certificate profiles with existing PKI practice.
X.509 PEM PKCS#12
